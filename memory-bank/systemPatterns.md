# System Patterns

This file documents recurring patterns and standards used in the project.

## Coding Patterns
*   **Component-Driven Development**: Build reusable UI components (Button, Card, Badge, Modal) using React and Tailwind CSS.
*   **Functional Components**: Use React functional components with hooks for state and side effects.
*   **Tailwind Utility-First**: Use Tailwind CSS for all styling to ensure consistency and rapid prototyping.

## Architectural Patterns
*   **Page-Based Routing**: Use Next.js App Router for navigation between the three main screens.
*   **Mock Data**: Use local JSON or constant files to store watch data, movement health stats, and vault inventory for the prototype.

## UI/UX Patterns (ArenaClub Inspired)
*   **Dark Mode/Luxury Aesthetic**: Use a dark, sophisticated color palette (blacks, deep grays, gold/accent colors).
*   **Card-Based Grid**: Marketplace items displayed in clean, uniform cards with prominent grading/health badges.
*   **Vault Centricity**: Emphasize that items are "in the vault" with specific UI treatments (e.g., vault icons, security badges).
*   **Interactive Dashboards**: Use charts or progress bars for Movement Health metrics.
